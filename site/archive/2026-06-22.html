LI . NO. 02
Three companies shipped the same idea this week without coordinating.

OpenRouter released Fusion: one API that fans your prompt out to multiple models simultaneously, a judge model picks the best response, you pay roughly half the price of a single frontier model.

Sakana AI released Fugu: a model trained to call other models. Recursively if needed. Single API, no single-vendor dependency.

DEPT released Deptify: an AI layer that routes work to whatever tools a client already has, connecting everything without replacing anything.

Different products. Same bet.

The bet is that the value isn't in the biggest model anymore. The value is in whatever knows which model to call.

Six months ago, "model router" was a niche builder concept. This week it shipped as a product strategy at three separate companies in the same seven-day window.

When the same answer shows up independently from three different directions, it's usually the right answer.

If you're building anything on AI right now, the routing layer is worth understanding. Because it's not background infrastructure anymore. It's becoming the product.

"We're figuring it out" starts to sound like a different answer than it did last year.

  • Sakana Fugu - Sakana AI Instead of training a bigger model, Sakana trained one model to call other models. Fugu orchestrates whatever combination of models fits the task, recursively if needed, through a single API. Their frame: orchestration models are the next frontier, not larger monoliths. Benchmarks claim frontier-level performance with no single-vendor lock. Worth watching because three companies made this same bet in the same week. https://sakana.ai/fugu-release/

  • Gemma 4 12B - Google / GADGETARQ Native audio input, no separate multimodal encoder, benchmarks close to the 26B model, runs on a laptop. The architectural move is what's interesting: they folded audio and image directly into the LLM backbone instead of bolting on a separate encoder. Cleaner, smaller. The Gemma family is at 150 million downloads and keeps getting more capable at smaller sizes. https://gadgetarq.com/introducing-gemma-4-12b-a-unified-encoder-free-multimodal-model/

  • Amazon Nova 2 + Nova Forge - Amazon Nova 2 ships Lite, Pro, a real-time voice model (Nova Sonic), and an experimental multimodal reasoning model (Nova Omni). The part most people will skip past is Nova Forge: take an unfinished base model, add your own training data, and build a custom AI for your specific workflows. AWS cloud clients building specialized tools are the target. The custom model play is getting more accessible across every major cloud. https://loscamposrv.com/article/amazon-launches-nova-ai-2-models-and-nova-forge-a-new-era-of-custom-ai

New harnesses & tools

  • OpenRouter Fusion - AI Daily Brief / OpenRouter You send a prompt, it fans out to multiple models in parallel, a judge model reads every response and picks the best fit for the job. OpenRouter says it achieves Fable-level intelligence at half the price. This is model routing going from niche builder concept to productized feature. For anyone running agents who's watched token costs compound on a single frontier model, it's worth testing. https://www.youtube.com/watch?v=VK56IPRsA-s

  • ValidMind Atryum - ValidMind / Open Source For You Open-source governance layer for AI agents. Sits directly in the execution path of every tool call and LLM call your agents make. The key distinction: it's not just checking whether an agent has permission to do something. It's checking whether the action makes sense for that agent's actual role and authority. As agents start writing to production systems and moving money, "has permission" stops being enough. https://www.opensourceforu.com/2026/06/validmind-launches-atryum-as-open-source-agent-governance-platform/

  • Tigera Lynx - Tigera / Open Source For You Unified control plane for Kubernetes-native AI agents. Every agent-to-agent, agent-to-tool, and agent-to-LLM interaction gets discovered, authenticated, authorized, and audited. No code changes to the agents themselves. Built on the Calico open-source security framework. If you're running agents at any scale on Kubernetes, the governance question is about to get louder and this is one answer. https://www.opensourceforu.com/2026/06/tigera-launches-lynx-to-govern-kubernetes-ai-agents/
